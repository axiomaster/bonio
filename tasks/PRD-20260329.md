# 产品需求文档 (PRD)：BoJi 桌面拟人助手

**文档版本:** V1.0
**产品定位:** 一款运行在移动设备全局桌面上，以语音交互为主、具备高度拟人化情绪表现与物理动态交互能力的 AI 桌面玩偶（默认形象为小猫）。
**核心价值:** 提供高感知的屏幕交互辅助、碎片化信息处理，以及深度的情绪陪伴价值。

---

## 1. 核心表现机制 (视觉与状态字典)

BoJi 在屏幕上具有实体形态，它不仅会切换动画，还会**在屏幕上物理移动**。它的任何行为都必须遵循以下五种基础状态的视觉规范：

| 状态名称 | 视觉与动作表现 (动画) | 交互与UI反馈 |
| :--- | :--- | :--- |
| **Idle (待机)** | 停留在屏幕边缘安静趴着，周期性播放呼吸、偶尔眨眼、摇尾巴的循环动画。 | 处于静默陪伴状态。偶尔头顶会冒出类似漫画风格的“纯文本文字气泡”（如：“今天天气真好喵”）。 |
| **Listening (聆听)** | 瞬间切换为专注状态，双耳竖起，眼睛睁大看向屏幕前方。 | 当用户点击 BoJi 触发语音输入时展现。麦克风开启，头顶无气泡，表示正在认真听用户说话。 |
| **Thinking (思考)** | 眼睛左右转动，或者头顶冒出转动的齿轮、问号，表现出“正在努力处理信息”的动作。 | 用户语音输入结束后展现。表示正在等待后端大脑的回复，缓解用户等待焦虑。 |
| **Speaking (回复)** | 坐正或站立，嘴巴配合语音节奏开合，前爪偶尔挥舞。 | 播放带有情感色彩的拟人语音。同时，头顶弹出【文字气泡】，同步显示正在说出的文字内容。 |
| **Working (长任务)**| **本体**恢复 Idle 状态或继续与用户聊天；**屏幕角落**凭空“啵”地弹出一个**缩小版的 BoJi 分身**（如戴眼镜敲键盘的猫），持续播放劳作动画。 | 当触发需要较长时间处理的任务时展现。任务完成后，分身动画消失，本体弹气泡语音播报结果。 |

## 2. 高价值用户场景详细需求 (User Scenarios)

### 场景一：智能来电守卫 (Call Handler)
**触发条件:** 用户手机收到系统来电。
**功能描述:** BoJi 自动接管来电提醒，智能识别来电身份，并替用户执行接听/挂断的物理操作。
* **分支 A：识别为骚扰/诈骗电话**
    1.  **预警:** BoJi 瞬间变为“红色警报”动画（全身散发红光或愤怒表情）。
    2.  **气泡与语音:** 头顶弹出红色气泡：“发现诈骗电话！10秒后自动挂断喵！”气泡内显示 `10...9...8` 的动态倒计时。同步语音播报。
    3.  **跑位:** BoJi 的实体在屏幕上快速跑到系统的“挂断”按钮旁边。
    4.  **执行:** 倒计时结束，或用户中途语音说“挂断它”，BoJi 播放一个“愤怒挥爪下拍”的动画，系统电话随之被挂断。BoJi 恢复正常颜色并回到屏幕边缘。
* **分支 B：正常来电**
    1.  **提醒:** BoJi 播放开心/活泼的动画。
    2.  **跑位与气泡:** BoJi 跑到系统的“接听”按钮旁边，头顶弹出绿色气泡：“主人，XXX 来电话啦，接不接喵？”显示 10 秒倒计时。
    3.  **执行:**
        * 用户语音说“接听”：BoJi 播放“开心按键”动画，电话接通。
        * 用户语音说“挂断”：BoJi 迅速跑到“挂断”按钮旁，播放“按键”动画，电话挂断。
        * 倒计时结束无响应：自动挂断。

### 场景二：通知消息智能“剪报” (Notification Secretary)
**触发条件:** 后台积累了多条未读通知，或用户主动询问“有什么新消息”。
**功能描述:** BoJi 自动阅读并理解杂乱的系统通知，过滤无用信息，以极简的口语化方式向用户汇报。
1.  **主动提示:** 如果有重要通知（如微信群长聊结束、快递送达），BoJi 本体头顶冒出“发光的灯泡”气泡，并发出“叮”的一声。
2.  **汇报表现:** 用户点击 BoJi 后，BoJi 说：“主人，我刚才瞥了一眼通知，帮您整理好啦。”
3.  **视觉呈现:** 屏幕中央展开一个精致的、类似“手账本/剪报”的 UI 面板（非气泡）。里面用大白话列出核心内容（例：“妈妈叫你今晚回家吃饭”、“拼多多有三个群在让你砍一刀，已忽略”）。

### 场景三：屏幕内容一键速记 (Screen Scouter)
**触发条件:** 用户在浏览任何 App（如小红书、大众点评、微信文章）时，按住 BoJi 并语音说出“记一下这个”或“帮我存下来”。
**功能描述:** BoJi 能够“看懂”当前屏幕上的内容，提取关键信息并自动整理保存。
1.  **接单:** BoJi 播放 Listening 动画，气泡显示“明白，我正在看屏幕喵！”
2.  **处理:** 切换为 Thinking 动画（闭眼转圈）。此时后台正在理解当前屏幕的画面文本。
3.  **反馈:** 处理完毕，BoJi 播放 Speaking 动画，气泡显示：“搞定！这家‘老王烧烤’的地址和推荐菜我已经写进您的备忘录啦！”

### 场景四：物理级自动代打字 (Keyboard Assistant)
**触发条件:** 用户在任何 App 的输入框界面，呼唤 BoJi 帮忙回复消息。
**功能描述:** BoJi 不仅生成文本，还会“跑到”输入框里，将文字一个个敲进去，提供极强的陪伴感。
1.  **跑位:** 用户语音说完指令后，BoJi 实体自动移动到当前屏幕焦点【输入框】的正上方或侧边。
2.  **敲击动画:** BoJi 切换为特制的“疯狂敲击键盘”动画。
3.  **文字输入:** 随着猫爪的挥舞，输入框内以人类打字的速度，逐字出现生成的回复文本。
4.  **完成:** 敲字结束，BoJi 擦擦汗（播放完成动画），回到屏幕边缘。

### 场景五：深夜健康“碎碎念” (Gentle Nagger)
**触发条件:** 达到特定时间（如凌晨 1 点）且用户仍在频繁滑动屏幕，或连续亮屏使用超过 2 小时。
**功能描述:** 提供情感价值，像家人一样对用户的健康进行干预。
1.  **疲惫状态:** BoJi 在屏幕边缘的 Idle 动画变为“睡眼惺忪、打哈欠、身体摇晃”的疲惫状态。
2.  **强制打断:** 如果用户继续使用，BoJi 会主动走到屏幕正中间，挡住部分视线。
3.  **语音抗议:** 播放严肃但可爱的语音：“还不睡觉！BoJi 的毛都要掉光了！快关掉手机睡觉喵！”头顶显示生气的气泡。
4.  **撤退:** 持续 5 秒后，BoJi 叹口气，走回屏幕边缘。

### 场景六：长文智能“代读” (Smart Reader Clone)
**触发条件:** 用户在长文章/长网页界面，告诉 BoJi “太长了，帮我总结一下”。
**功能描述:** 利用“分身”机制处理耗时阅读，本体直接汇报结果。
1.  **派发任务:** BoJi 本体说：“没问题，我让小弟去读。”
2.  **分身出现:** 屏幕右上角弹出一个“戴眼镜、疯狂翻书/看屏幕”的微型 BoJi 分身（Working 状态）。
3.  **本体汇报:** 几秒钟后，本体 BoJi 开始用语音流畅地播报长文的 3 句话核心摘要。
4.  **任务结束:** 播报完毕，右上角的微型分身化作一缕烟消失。

## 3. 全局交互设计规范 (Interaction Guidelines)

* **物理沉浸感:** BoJi 绝不是死板地固定在一个位置。当需要与屏幕特定元素（如按钮、输入框）交互时，它必须计算坐标，并带有**平滑移动（Translate）过渡**，走到目标位置后再执行动作。
* **不打扰原则:** 在 Idle（待机）状态下，BoJi 必须安静地待在屏幕边缘（用户可自由拖拽它的默认栖息位置），绝不可阻挡用户的正常滑动和点击操作。
* **状态互斥与打断:** 当 BoJi 正在 Speaking（说话）或播放耗时动画时，用户如果再次点击 BoJi 的身体，必须能够立即打断当前行为，强制进入 Listening（聆听）状态。
* **气泡的呼吸感:** 文字气泡的出现和消失应当带有平滑的淡入淡出（Fade in/out）和轻微的弹性缩放（Spring Scale）动画，拒绝生硬的闪现。文字内容需根据语速动态展现。
