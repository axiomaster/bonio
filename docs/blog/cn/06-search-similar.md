# 搜同款：一键比价的购物搭子

> 看到一张好看的商品图，想知道它多少钱、在哪买？圈一下，Bonio 帮你搜遍全网。

---

## 场景：从"看到"到"买到"

你在刷网页、看视频、翻社交媒体，突然看到一个喜欢的商品——一件衣服、一款耳机、一把键盘。你想要比价、看评价、找购买链接。传统流程是：

1. 截图或保存图片
2. 打开淘宝（或京东、拼多多）
3. 找到"以图搜图"入口
4. 上传图片
5. 等待结果

"搜同款"把这个流程压缩成：**右键 → 搜同款 → 圈一下 → 看到结果。**

---

## 交互流程

**第一步：触发。** 用户在 Avatar 上右键，选择"搜同款"。此时 Bonio 进入 **Lens 标注模式**：

- Avatar 窗口扩展到覆盖整个锚定窗口
- 窗口周围出现 3px 红色边框，表示"已捕获，请标注"
- 光标变为十字准星 (`SystemMouseCursors.precise`)
- 工具栏出现在 Avatar 旁边：**取消** / **撤回** / **确认**

**第二步：圈选。** 用户按住左键拖拽，在商品图上画一个红色矩形框。可以画多个框（如果页面上有多个商品）。点"撤回"删掉上一个框，重新画。

**第三步：搜索。** 点击"确认"。Bonio：

1. 裁剪出圈选区域的高清截图（PNG 编码）
2. Avatar 移动到圈选区下方，播放**搜索动画**（thinking 状态，头顶冒出放大镜气泡）
3. 打开一个 WebView 窗口
4. 自动导航到淘宝图片搜索页（`https://www.taobao.com`）
5. JS 注入：自动点击上传按钮，填入裁剪图片，触发搜索
6. WebView 定位在 Avatar 下方，像一个"从 Avatar 身上弹出"的结果面板

**第四步：浏览。** 用户在新弹出的 WebView 窗口中查看淘宝以图搜图的结果——同款商品、价格区间、购买链接，一应俱全。

## 技术实现

### Lens 标注复用

"搜同款"的圈选交互复用 Bonio 的 **AI Lens** 子系统——与"圈一圈"（AI 视觉问答）共享同一套窗口标注引擎：

```
Lens 标注覆盖层
├─ 全窗口截图（PrintWindow API，PW_RENDERFULLCONTENT）
├─ 红色边框指示器（3px，表示标注模式激活）
├─ 十字准星光标
├─ 多矩形标注（GestureDetector + 坐标转换）
├─ 工具栏：取消 / 撤回 / 确认
└─ 坐标系统：
    ├─ 标注坐标：相对于窗口左上角
    └─ 发送坐标：转换为截图物理像素（× DPI scale）
```

AI Lens 是 Bonio 的通用视觉标注基础设施。"搜同款"和"圈一圈"都是它的上层应用——前者将标注结果用于以图搜图，后者用于 AI 视觉问答。

### 淘宝搜图：WebView + JS 注入

为什么不调用商业搜图 API？

| 方案 | 成本 | 覆盖率 | 中文电商适配 |
|------|------|--------|------------|
| Google Vision API | $1.50/千次 | 全球 | 一般 |
| 淘宝开放平台 API | 需企业资质 | 淘宝内 | 优秀 |
| **WebView + 淘宝网页版** | **零成本** | **全量商品** | **完美** |

Bonio 选择的方案：**直接使用淘宝网页版的以图搜图功能**。淘宝的网页版以图搜图不需要登录、不受频率限制、覆盖全量商品——唯一的"缺点"是它原本设计给人手动操作的。Bonio 通过 WebView + JS 注入让它变得可编程：

```javascript
// 注入到淘宝图片搜索页的 JS（简化）
// 1. 找到隐藏的文件上传 input
const uploadInput = document.querySelector('input[type="file"]');

// 2. 将裁剪的图片数据构造为 File 对象，赋值给 input
const blob = base64ToBlob(croppedImageBase64, 'image/png');
const file = new File([blob], 'search.png', { type: 'image/png' });
const dt = new DataTransfer();
dt.items.add(file);
uploadInput.files = dt.files;

// 3. 触发 change 事件，让页面自动上传并搜索
uploadInput.dispatchEvent(new Event('change', { bubbles: true }));
```

用户在 WebView 中看到的就是淘宝的正常搜索结果页——图片、价格、店铺、链接。这是完完全全的真实淘宝体验，Bonio 只是帮你省掉了"打开淘宝 → 找到搜图入口 → 上传图片"这个过程。

### WebView 窗口管理

搜索结果展示在一个**独立的 WebView 窗口**中（通过 `desktop_multi_window` 创建）。这个窗口：

- 初始大小 420×680（适合商品搜索结果浏览）
- 定位在 Avatar 下方，视觉上像是"从 Avatar 身上弹出来的"
- 用户可自由拖拽、调整大小、关闭
- 关闭后 Avatar 恢复待机状态

## 为什么叫"搭子"？

"搭子"是当代年轻人的社交用语——饭搭子、运动搭子、旅游搭子。它比"朋友"轻，比"工具"暖，是一种**特定场景下的陪伴关系**。

"购物搭子"就是你在购物时需要的那个人——帮你搜、帮你比价、帮你找同款。Bonio 的"购物搭子"做不了复杂的消费决策，但它让"看到 → 搜到 → 买到"这个流程几乎不费力。

这正是 Bonio 的产品定位：**它不替代你的判断，而是消除你做判断之前的摩擦。**

---

## 隐私考量

"搜同款"功能有几个刻意的隐私设计：

1. **你圈什么就搜什么。** 不会自动上传整张截图——只会裁剪你手动圈选的区域。没有圈选就没有上传。
2. **图片数据直传淘宝。** Bonio 服务器不中转你的图片。WebView 内的 JS 直接构造 FormData 提交给淘宝的搜索接口。
3. **不走 Bonio 后端。** 整个搜同款流程中，图片不经过 HiClaw/OpenClaw 服务端。这是客户端本地操作。
