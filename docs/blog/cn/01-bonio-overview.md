# Bonio 总览：桌面 AI 伴侣的新范式

> 它不是一个聊天窗口，而是一个坐在你窗口边缘的、有灵魂的小家伙。

---

## 什么是 Bonio？

Bonio（波妞）是一个运行在你 Windows 或 macOS 桌面上的 **AI 伴随智能体**。它不是藏在任务栏里的后台服务，也不是需要你切过去才能对话的聊天窗口——它是一只浮动在你当前窗口边缘的 **虚拟宠物**，看得见你的屏幕，听得见你的声音，能在你需要时默默帮忙，也能在你无聊时逗你开心。

从技术上说，Bonio 是一个**多平台、多引擎、插件化的桌面 AI 代理系统**。它由四个组件构成：

| 组件 | 技术栈 | 角色 |
|------|--------|------|
| **Desktop 客户端** | Flutter / Dart | Windows & macOS 桌面端，承载 Avatar 渲染、GUI 自动化、语音交互 |
| **Server（HiClaw）** | C++17 / CMake | 自研网关服务器，WebSocket 协议，AI Agent 编排 |
| **Android 客户端** | Kotlin / Jetpack Compose | 手机端设备控制（相机、定位、短信等） |
| **HarmonyOS 客户端** | ArkTS | 鸿蒙端，与 Android 端功能对等 |

这些组件通过统一的 **WebSocket 网关协议**（v3）通信，无论客户端跑在哪个平台，都能共享同一套后端 AI 能力。

## 理念：AI 不应该只是一个聊天框

过去两年，AI 助手的主流形态一直是"对话式界面"——一个输入框、一段回复、来回往复。这很高效，但缺少一样东西：**存在感**。

Bonio 的设计哲学完全不同：

1. **AI 应该主动陪伴，而非被动等待召唤。** Bonio 浮动在你的窗口边缘，跟随你切换应用，像一个坐在你肩膀上的小助手。它有自己的"情绪"和"动作"，会随机踱步、打哈欠、伸懒腰——不是无聊的装饰，而是让你感知到"它在那里"的存在设计。

2. **AI 应该感知你的上下文，而非每次从零开始。** Bonio 知道你当前在用哪个应用、窗口标题是什么、浏览器里打开的是什么网页。你把文件拖给它，它就知道你想"记下来"；你在浏览器里看文章，它就问你要不要"伴读"。

3. **AI 应该能操控你的桌面，而非只停留在文本里。** Bonio 可以截图、调整窗口布局、在浏览器里执行 JavaScript、通过 CDP 协议读取页面内容。它是一个真正的 **GUI Agent**，而不只是一个 LLM wrapper。

4. **AI 应该是可扩展的生态，而非封闭的功能集。** 通过插件系统，任何开发者都能用任意语言为 Bonio 添加右键菜单能力。比价、记笔记、伴读——这些功能本身就是插件，你也可以写自己的。

## 功能全景

### 🐱 Avatar 系统
你会看到一只猫（或你选择的主题角色）浮动在当前窗口的顶边。它可以：
- 跟随窗口移动和切换，带弹簧物理感的平滑追踪
- 响应 5 种交互：单击（随机动作+气泡）、双击（文本输入）、长按（语音输入）、拖拽（手动移位）、右键（功能菜单）
- 展示 15+ 种动画状态：待机、思考、说话、倾听、行走、奔跑、拖拽、敲击、拍照……
- 通过 Lottie 动画 + 主题系统实现完全可替换的外观

### 🧠 AI 对话
双击 Avatar 或打开主窗口，就能与 AI 对话。支持：
- 流式响应，实时看到 AI "打字"
- Tool Call 可视化（你能看到 AI 在调用什么工具）
- 多模型切换，支持思考级别调节（off/low/medium/high）
- 图片附件、会话历史、多会话管理

### 🖱️ 右键工具箱
在 Avatar 身上右键，弹出上下文菜单。当前内置：
- **记一记**：截图当前窗口 → AI 自动分析、打标签、写摘要 → 存入记忆
- **圈一圈**：进入标注模式，在窗口上画红框 → AI 聚焦分析标注区域
- **搜同款**：圈选商品图片 → 自动跳转淘宝以图搜图
- **伴读**：浏览器中长文阅读 → 70/30 分屏 → 自动提取目录+摘要+笔记编辑器

这些功能的背后是一个**插件系统**——它们本身就是插件。第三方开发者可以用任意语言（Dart、Python、Go、Node.js）开发新插件，通过 JSON-RPC 与宿主通信。

### 🎤 语音交互
长按 Avatar 开始说话，松开发送。背后的技术栈：
- **STT（语音识别）**：Sherpa-ONNX 流式 paraformer，中英双语，完全本地运行
- **TTS（语音合成）**：Windows 用 SAPI，macOS 用 `say`，Linux 用 espeak
- **意图路由**：服务端自动分类语音输入——是闲聊、是截屏指令、是来电接听/挂断？

### 📱 微信远程指挥
通过企业微信（WeCom）或个人微信（ilink），你可以用手机给 Bonio 发消息。问它问题、让它执行任务——它会在你的电脑上响应。你在地铁上想起来要让电脑做件事，掏出微信说一声就好。

### 🧿 记忆系统
Bonio 有自己的"记忆"。你可以通过右键"记一记"保存窗口截图，也可以直接把文件、图片、文本拖到它身上"投喂"给它。AI 会自动分析内容、打标签、写摘要，存入本地文件系统。以后你可以问它："帮我把所有 #购物 相关的笔记找出来"。

### 🔌 双引擎后端
Bonio 的桌面端可以连接两种后端：
- **HiClaw**（自研 C++ 服务）：支持 Skill 系统、Provider 系统、微信集成、Cron 定时任务、Memory 管理
- **OpenClaw**（社区 Node.js 服务）：公共网关，开箱即用

两者遵循**同一套 WebSocket 协议**，客户端无需关心后端实现。

## 为什么用 Flutter？为什么用 C++？

**Desktop 选择 Flutter**，因为它能一套代码覆盖 Windows 和 macOS。对于 Avatar 这种需要透明窗口、自绘动画的场景，Flutter 的渲染引擎比 Electron 轻得多，比原生开发快得多。

**Server 选择 C++**，因为我们追求极致的部署便利性。HiClaw 编译出来是一个 **单个可执行文件**（约 15MB），零依赖，拷贝即运行。不需要 Python 环境、不需要 Node.js、不需要 Docker。这在私有部署场景中至关重要——用户只想双击 exe，不想折腾环境。

## 开源与社区

Bonio 是一个开源项目，代码托管在 GitHub。它的设计深受 OpenClaw 和 ZeroClaw 的启发——这两个项目是手机端 AI Agent 的先驱，Bonio 将它们的概念带到了桌面端，并拓展了 Avatar、微信、插件等全新维度。

我们相信：**桌面 AI 的终极形态不是 Copilot 式的侧边栏，而是一个有存在感、有记忆、能动手的"数字搭子"。**
